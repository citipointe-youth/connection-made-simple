====================================================
These cannot be done in code — they require manual steps in external dashboards.

1. SET SESSION_SECRET IN VERCEL
   Without this, session tokens can be forged by anyone who knows the fallback value.

   Steps:
   a) Generate a secret — run this in a terminal:
      node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
   b) Vercel dashboard → Project → Settings → Environment Variables
   c) Add: SESSION_SECRET = <the generated value>
      Set scope: Production (and Preview if you use preview deployments)
   d) Redeploy: Vercel dashboard → Deployments → Redeploy latest

2. APPLY MIGRATION 003 IN SUPABASE
   Fixes: settings table UUID type mismatch that will crash the app on first settings load.
   Also enforces a single settings row (prevents concurrent cold-start race condition).

   Steps:
   a) Supabase dashboard → Project → SQL Editor
   b) Paste and run the full contents of:
      supabase/migrations/003_settings_singleton.sql
   c) Verify it ran without errors

3. LOCK CORS TO YOUR DOMAIN
   Currently accepts API requests from any website. Should be locked to your domain.

   Steps:
   a) Vercel dashboard → Project → Settings → Environment Variables
   b) Add: CORS_ORIGINS = https://connection-made-simple.vercel.app
   c) Redeploy

4. PASSWORD HASHING UPGRADED TO BCRYPT ✓
   hashPassword() now uses bcryptjs (12 rounds, ~300ms).
   verifyPassword() auto-detects legacy SHA-256 hashes and handles them
   correctly. On first successful login with a legacy hash, the password
   is silently re-hashed to bcrypt and saved — no user action needed.
   New passwords (set via Admin → Accounts) are always bcrypt.

5. SUPABASE ROW-LEVEL SECURITY ENABLED ✓
   Migration 006 enables RLS on all 13 tables with no anon policies,
   so any connection via the Supabase anon key (PostgREST) is denied.
   The Express API uses the postgres superuser connection string which
   bypasses RLS automatically — no impact on app behaviour.
   Apply migration 006 in Supabase SQL Editor to activate.
